Git Workflow Guide

Step-by-step guide to initialize a Git repository, manage files, and push changes to a remote repository.

  1. Download and install Git
  2. Create a work folder on your PC for Git
  3. Initialize a Git repository in your folder by running this command in the terminal: git init
  4. Create a remote repository on GitHub, GitLab, or use an existing one
  5. (Optional) Create a branch if no default branch is configured: git branch my-new-branch
  6. (Optional) Switch to the created branch: git checkout my-new-branch
  7. Connect your local folder to the remote repository: git remote add origin "https://github.com/username/reponame.git"
  8. Confirm the connection: git remote -v
  9. Add and modify files locally
  10. Add those files or changes to Git: git add --all
  11. Commit the changes: git commit -m "commit message"
  12. (Optional) Add tags where necessary: git tag <tagname>
  13. Confirm your changes:
    • git log
    • git log --oneline
    • git rev-parse <branch-name>
    • git show --name-only
    • git log --oneline --decorate --graph --all
    • git log --oneline --graph
  14. Switch branch to the main branch (usually: master or main - but can be changed based on the default settings). Therefore change the main to master or to the default branch: git branch -M main
  15. Push changes to remote: git push -u origin main
  16. (Optional) Push changes to a different branch: git push -u origin <branch-name>

QUICK NOTE:

All files are untracked until git add is performed on them, after which they become tracked files.

More Git Operations

Git Stash

Save uncommitted changes and return later.

  • Stash your changes: git stash
  • Stash with a message: git stash push -m "stash message"
  • List all stashes: git stash list
  • Create a branch from a stash: git stash branch <branchname> 'stash@{0}'
  • Show stash details: git stash show
  • Apply a specific stash (stash remains in stack): git stash apply 'stash@{n}'
  • Apply latest stash and remove it from stack: git stash pop
  • Delete a specific stash: git stash drop 'stash@{0}'
  • Clear all stashes: git stash clear
  • Create a new branch and apply a stash: git stash branch

Git Tag

Label or bookmark a specific commit.

  • Create a lightweight tag: git tag <tagname>
  • Create an annotated tag: git tag -a <tagname> -m "message"
  • Tag a specific commit: git tag <tagname> <commit-hash>
  • List tags: git tag
  • Show tag details: git show <tagname>
  • Push tag to remote: git push origin <tagname>
  • Push all tags: git push --tags
  • Delete a local tag: git tag -d <tagname>
  • Delete a remote tag: git push origin --delete tag v1.0
  • Update or replace a local tag: git tag -f <tagname> <new-commit-hash>
  • Update or replace a remote tag: git push --force origin <tagname>

Git Branch & Other Useful Commands

Manage branches, rename, and inspect branches.

  • Delete a local branch: git branch -d branch-name
  • Delete a remote branch: git push origin --delete branch-name
  • Rename a branch: git branch -m old-name new-name
  • Show remote branches: git branch -r
  • Show all branches (local + remote): git branch -a

Amend

Modify the most recent commit – fix typos, add/remove files, or change the commit message.

  • Fix last commit message: git commit --amend -m "New message"
  • Add file to last commit: git add <file> then git commit --amend
  • Add file to last commit without changing the commit message: git add <file> then git commit --amend
  • Remove file from last commit: git reset HEAD^ -- <file> then git commit --amend

Rebase

Combine a sequence of commits onto a new base commit.

  • Reapply feature branch changes on main: git checkout feature-branch then git rebase main
  • Interactive rebase: git rebase -i
  • Edit, reorder, squash, fix commits: git rebase -i HEAD~3
  • Continue rebase after conflicts: git rebase --continue
  • Abort rebase: git rebase --abort
  • Skip a commit during rebase: git rebase --skip

Revert

Create a new commit that reverses changes.

  • Revert latest commit: git revert HEAD
  • Revert specific commit: git revert <commit-hash>
  • Revert commit further back: git revert HEAD~2
  • Skip commit message editor: git revert --no-edit
  • Show commit history to find commits to revert: git log --oneline

Recovery / Restore

Recover deleted branches, lost commits, or files.

  • Recover lost branch: git reflog then git checkout -b branch-name <commit-hash>
  • Restore a file from latest commit: git restore filename.txt

Reset ⚠️⚠️⚠️

Move your current branch (HEAD) to a different commit.

  • Move HEAD but keep changes staged: git reset --soft <commit>
  • Move HEAD, unstage changes: git reset --mixed <commit>
  • Move HEAD and discard all changes: git reset --hard <commit>
  • Unstage a file: git reset <file>
  • Show commit history: git log --oneline

Reflog ⚠️⚠️⚠️

Recover lost commits, undo resets, and see history of branch & HEAD.

  • View reflog: git reflog
  • Reset to a specific reflog state: git reflog then git reset --hard HEAD@{2}
  • Expire old reflog entries and garbage collect: git reflog expire --expire=30.days refs/heads/main then git gc --prune=now
